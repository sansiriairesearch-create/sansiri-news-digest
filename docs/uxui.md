## UX-03 — News Item Component (ต่อข่าว)

โครงสร้าง 3 ส่วน:
1. **พาดหัว** — `<a>` 15px **bold**, สี `#000000`, **`text-decoration:none`** (ไม่ขีดเส้นใต้ — เช็ก C7), ลิงก์ไปแหล่งที่ดีที่สุด
2. **แถวลิงก์แหล่งข่าว** — 13px สี `#0563C1` ขีดเส้นใต้ เรียง **print → online → social** (ภายในแต่ละชนิดเรียง PR Value มาก→น้อย)
   - แสดง **ชื่อสื่อเฉย ๆ** ห้ามใส่ป้าย "(print)" / "[PDF]" / "(FB)" (เช็ก C2 · [PRD.md BR-05](./PRD.md))
   - **ลิงก์ online/social ต้องชี้ตรงหน้าข่าวจริง** — ถอด prefix `https://re.dataxet.co/api/` และซ่อม `https:/`→`https://` (เช็ก C10 · [PRD.md BR-08](./PRD.md)) เพื่อไม่ให้ Gmail Redirect Notice เด้งไป URL ของ Dataxet
   - print link (ไฟล์ PDF ข่าวตัด `iqnewsclip.com/reportdl/printdl`) **คง** prefix `re.dataxet.co/api/` ไว้ และอยู่ลำดับแรก
3. **สรุป** — 14px สี `#000000` line-height 1.7, ความยาว 2–4 ย่อหน้า ([PRD.md BR-04](./PRD.md))

ทุกข่าวต้องมีลิงก์แหล่งข่าวอย่างน้อย 1 (เช็ก C8); URL ต้องไม่เพี้ยน `https://https:/` (เช็ก C1)

## UX-04 — Subject Line & Date Formatting

- **Subject (รูปแบบอ้างอิงจากงานที่ทำด้วยมือ):**
  `ข่าวเกี่ยวกับกลุ่มบริษัทแสนสิริ ข่าวคู่แข่งและอสังหาทั่วไป  วันที่  DD  MMMM  YYYY(พ.ศ.)   มีรายละเอียดที่น่าสนใจดังนี้`
- **วันที่ในตัวอีเมล:**
  - intro + subject ใช้ **ภาษาไทย + พ.ศ.** เช่น `24 มิถุนายน 2569`
  - subtitle บน banner ใช้ **อังกฤษ + ค.ศ.** เช่น `June 24, 2026`
- อ้างอิงเวลา **Asia/Bangkok** เสมอ ([PRD.md NFR-02](./PRD.md))

> หมายเหตุ: subject ของ "อีเมลตาราง" ที่ระบบ GAS ส่ง (`Sansiri News (DD MON YY / HH:MM)`) เป็นคนละผลิตภัณฑ์ — อยู่นอกขอบเขต

## UX-06 — Gmail Rendering Constraints

- **Inline CSS เท่านั้น** — ห้ามใช้ `<style>…</style>` (เช็ก C11)
- **Table-based layout** เพื่อรองรับ Gmail + Outlook
- **ห้ามฟอนต์ Noto Sans Thai** (เช็ก C12) — ใช้ GraphikTH stack เท่านั้น
- เคารพ **body cap ~102 KB** ของ Gmail (สรุปกระชับ; ไม่ฝังรูปใหญ่)
- footer ไม่มีสีพื้นหลัง (พื้นขาว)

## UX-06b — Draft preview vs. sent email (สำคัญ)

อาการสองอย่างนี้เป็น **artifact ของ "ฉบับร่าง/หน้าจอแก้ไข" (compose view) เท่านั้น** จะหายเองเมื่ออีเมลถูกส่งจริง:

| อาการในหน้า draft | สาเหตุ | ในอีเมลที่ส่งจริง |
|---|---|---|
| Banner สีน้ำเงิน/ทองหายไป ("Sansiri News" ขาวมองไม่เห็น) | compose view ของ Gmail ตัด `background-color`/`bgcolor` ทิ้งตอนแก้ไข | แสดงสีพื้น banner ปกติ |
| คลิกลิงก์แล้วขึ้นหน้า **"Redirect Notice"** ของ Google | URL ของ draft ยังไม่มีลายเซ็น `usg=` (Gmail เติมให้ตอน "ส่ง" เท่านั้น) | เด้งเข้าหน้าข่าวจริงทันที ไม่มี interstitial |

**วิธียืนยันของจริง:** เปิด draft (ที่ส่งถึงตัวเอง) แล้วกด **Send** → เปิดอีเมลที่เข้ามาในกล่อง Inbox จะเห็น banner สีครบและคลิกลิงก์เข้าหน้าข่าวตรง ๆ (Gmail MCP ไม่มี tool ส่งอีเมล จึงทดสอบ "ส่ง" ให้อัตโนมัติไม่ได้ ต้องกดส่งเอง)
